Delivery conditions

A phase ends when the next phase can begin.

Each stage has an explicit handoff. This prevents premature building, unclear ownership, and deliverables that cannot be implemented.

Research → Structure

Material questions are answered, assigned, or bounded.

Structure → Build

Ownership, flow, controls, and output requirements are clear.

Build → Implement

The system is usable, testable, editable, and documented.

Implement → Improve

Owners, feedback loops, and revision logic are active.
